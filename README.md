# README.md — 论文阅读自动化（中文说明）

## 项目标识

- **仓库地址**: `git@github.com:DavidCaoResearch/Paper-Reading-Template.git`
- **核心任务**: 学术论文的自动化阅读、分类与笔记撰写。
- **语言约定**: 笔记与文档使用中文；代码/shell/命令使用英文。

## 新设备一次性配置

在任何新机器上依次执行以下命令即可就绪：

### 1. 克隆与环境

```bash
git clone git@github.com:DavidCaoResearch/Paper-Reading-Template.git
cd Paper-Reading-Template
conda create -n your-env python=3.11 -y
conda activate your-env
pip install PyPDF2
```

### 2. Claude Code CLI

```bash
npm install -g @anthropic-ai/claude-code
```

### 3. Python 依赖

| 包名 | 版本 | 用途 |
|---|---|---|
| `PyPDF2` | ≥3.0 | PDF 标题提取与文本阅读 |

安装：`conda activate your-env && pip install PyPDF2`

### 4. Windows 特别说明

- **硬链接代替软链接**：Windows 创建符号链接需要管理员权限。本项目使用**硬链接**（Python `os.link()`）——效果同等去重，无需提权。
- **Conda 激活**：Windows cmd/PowerShell 中需分步执行 `conda activate your-env`，不能与后续命令用 `&&` 串联（仅 Git Bash 支持）。
- **开发者模式**：若需真正软链接，在 Windows 设置 → 隐私与安全性 → 开发者选项中启用开发者模式。

### 5. Watchdog 权限

Watchdog 使用 `claude -p --permission-mode bypassPermissions` 调用 Claude Code。这是自动化执行文件操作和 git 命令所必需的。Watchdog 仅在你信任的目录（你自己的仓库）中运行。

## 工作流程

### 1. 录入新论文

当 PDF 被放入 `原始文献/`（尚未归入子文件夹）时：

1. 在 `原始文献/` 下以论文标题新建子文件夹。
2. 将 PDF 移入该子文件夹，重命名为 `<论文标题>.pdf`。
3. **完整通读**论文内容——切忌略读。提取所有数学公式、算法步骤和实验数据。
4. 在同一子文件夹中按 §阅读笔记规范 创建 `notes.md`。
5. 确定分类——参见 §分类规则。
6. 更新 `原始文献/README.md` 中的总目录。
7. 提交并推送——参见 §提交与推送。

### 2. 分类规则

对每篇论文，根据以下维度确定 **1~N 个分类文件夹**：
- **研究问题领域**（如 "Rolling Stock Scheduling"）
- **使用方法论**（如 "Branch and Bound"、"Reinforcement Learning"）
- **子问题或背景**（如 "Train Platforming/Shunting"）

**规则：**
- **复用已有文件夹**：当匹配的分类已存在时直接使用。
- **新建文件夹**：当论文引入了尚未涵盖的新主题/方法时创建。
- 一篇论文可归属多个分类——在每个相关文件夹中创建硬链接。
- 硬链接指向 `原始文献/<论文标题>/` 中的 PDF，避免多重拷贝。

**Windows 下创建硬链接（无需管理员权限）：**
```python
import os
os.link("<源 PDF 绝对路径>", "<分类>/<论文标题>.pdf")
```

### 3. 阅读笔记规范 (`notes.md`)

每份 `notes.md` 必须**详细且具体**，非高层概览。每个章节须以**英文原文直接引用**作为支撑。

#### 3.1 核心内容

##### 研究问题
- 具体解决什么问题？在什么场景下（战术层/操作层、中断/正常）？难点在哪？
- 工业合作方 / 数据来源是谁？

##### 建模思路
**须包含以下高粒度信息：**
- **决策变量**：列出关键变量族，附数学符号与含义（如 `x_t_c ∈ {0,1}` —— 车次 t 是否分配编组 c）
- **约束条件**：列举主要约束组及其作用（如编组唯一性、流量守恒、库存平衡、LIFO 排序）
- **目标函数**：逐项列出，附权重系数与惩罚含义——不得仅说"加权求和"
- **模型规模**：变量数、约束数、单元类型数、车次数、车站数——引用确切数字
- **模型类型**：明确标注 MILP、IP、CP、图论、列生成等

> **目标**：读完本节后，研究者应能理解模型的数学结构而无需打开原论文。若论文发表了 MIP 公式，须引用约束方程。

##### 核心方法论
**须包含以下高粒度信息：**
- 算法名称、类型（精确/启发式/混合）、整体流程
- 关键子例程及其输入/输出
- 如有分解：分离/割平面逻辑
- 如有列生成：定价问题、dominance 规则
- 使用的商业求解器（CPLEX、Gurobi 等）及版本
- 参数配置：时间限制、MIP gap 容差、分支规则

> **目标**：读完本节后，研究者应能画出算法的伪代码。

##### 案例规模
**须包含具体数字：**
- 实例数量、来源（真实/人工）、时刻表周期
- 车队规模、单元类型数、车辆段数、车站数
- 车次数、matching 数、每条车辆段轨道数
- 关键参数值（车头时距、最大延迟等）
- **实验结果**：每个实例的运行时间、optimality gap、相对基准的改进百分比——引用表格中的具体数值

#### 3.2 核心创新点
- 逐条列出创新点，附原文引用。
- 区分*真正原创*的贡献（首次提出、范式转换）与*增量式*改进（扩展、参数调优）。
- 对理论贡献（如修正前人错误），包含反例逻辑。

#### 3.3 文献关联
- 与 `原始文献/` 中已有论文的对比/关联。要具体：共享基座模型、不同目标函数、互补子问题等。
- 标注共享的 benchmark、竞争方法或互补方法。
- 梳理引用谱系（哪些论文在哪些基础上构建）。
- 若无相关文献，写"暂无相关文献"，但仍列出与当前馆藏范围重叠的引用文献。

#### 3.4 写作手法
- 叙事结构、论证流程、可视化风格等值得学习的写作技巧。
- 值得效仿的表格/图表设计。
- 引用具体段落、图、表作为范例。

#### 数学公式格式

引用或描述数学公式（决策变量、约束条件、目标函数等）时，须使用 Markdown LaTeX 数学符号：
- 行内：`$x_t_c \in \{0,1\}$` → $x_t_c \in \{0,1\}$
- 独立行：`$$\sum_{c \in C} x_t_c = 1 \quad \forall t \in T$$`

#### 引用规则

第 3.1–3.4 节中的所有论断**必须**以论文英文原文的直接引用为支撑。使用引用块格式，标注页码/章节位置。每个子节至少一条引用。

### 4. 总目录

`原始文献/README.md` 是持续更新的索引，结构如下：

```markdown
# 原始文献总目录

## <分类名称>
- [<论文标题>](./<论文标题>/notes.md) — 一句话简述
```

每次录入或重新分类论文后更新该文件。分类按字母顺序排列。

### 5. 提交与推送

**每总结完一篇论文**，立即提交并推送：

```bash
git add -A
git commit -m "<前缀>: <描述>"
git push origin main
```

**提交信息前缀**（遵循 Conventional Commits 规范）：
- `add:` — 新论文录入及笔记
- `fix:` — 修正笔记、分类或链接
- `update:` — 增强已有笔记或索引
- `refactor:` — 重命名、重组文件夹结构
- `docs:` — 仅文档变更（CLAUDE.md、README.md）

**更新日志**: 每次 push 在 `更新日志/changelog.md` 中追加一条：
```markdown
## YYYY-MM-DD
- <变更描述>
```

### 6. 记忆管理

为满足跨设备使用需求，将项目持久化知识以 markdown 文件形式存储在 `memory/` 目录下。使用标准 memory 格式（含 `name`、`description`、`metadata` 前置元数据）。记忆文件记录：
- 用户偏好与约定
- 分类决策及其理由
- 项目级模式与经验教训

记忆文件与项目其他文件一并推送。

## 自动化录入（Watchdog）

**推荐用法（一次性扫描，默认）：**

1. 将一篇或多篇 PDF 拖入 `原始文献/`
2. 运行：`conda activate your-env && python watchdog.py`
3. 所有待处理论文依次处理完毕，脚本自动退出
4. 查看 `watchdog.log` 确认结果

此模式不做后台轮询——触发一次即退出，不会持续占用资源。

**持续监控模式**（快速连续添加论文时使用）：

```bash
conda activate your-env && python watchdog.py --watch
```

每 5 秒轮询一次，`Ctrl+C` 停止。

### Watchdog 工作原理

1. 扫描 `原始文献/` 顶层 PDF
2. 通过 PyPDF2 提取论文标题（元数据 → 首页文本 → 文件名回退）
3. 计算 SHA256 哈希做去重（相同论文不会重复处理）
4. 在项目目录下调用 `claude -p --permission-mode bypassPermissions`
5. 显示实时进度（旋转动画 + 跳动计时）
