# User Preferences

- **Language**: Notes and documentation in Chinese; commands and code in English.
- **Workflow**: Fully automated — once a PDF is placed in `原始文献/`, the entire pipeline runs without further input.
- **Cross-device**: This project is used across multiple devices. All state must be persisted in the git repo (no local-only assumptions).
- **Commit style**: Conventional Commits prefixes (`add:`, `fix:`, `update:`, `refactor:`, `docs:`).
- **Evidence standard**: All reading note claims must be supported by direct English quotations from the original paper.
- **Classification philosophy**: When uncertain, create more classifications — symlinks are cheap, missed connections are not.
- **Push cadence**: Push after every single paper is summarized; never batch multiple papers in one commit.
